Quick Start

Last updated: 

Installation

To get started with penguin.js is easy because it embraces the filesystem as its API. So there little need to fiddle in config files to just get-started out with your new project.

1. Setup your website folder with penguin.js

  • Create a new folder named like your project
    mkdir myawesomeness
  • Create a package.json with minimal markup
    {
  "penguin": {
    "languages": ["en"]
  },
  "scripts": {
    "serve": "penguin serve -v pug",
    "prebuild": "penguin pack -v pug",
    "build": "penguin build"
  }
}
  • Install penguin.js (make sure, you got at least node v6 installed)
    npm i -S penguin.js

2. Start your development server

npm run serve

Alright now if you got everything right, your development server should start and listen on port 3000.

Pages

Create your first page

If you visit your website on http://localhost:3000 you'll see a default 404 page. It's nice to see how fast you can start a new website with penguin but it's not yet useful. So let's create a first page.

  • Create pages/ directory
    mkdir -p pages
  • Create a file called index.pug in your pages directory and add some content.
    doctype html
html
  head
    title My first page
  body
    p My first content

A quick note on pug

Penguin.js is designed to be used with many template languages. But currently it only supports pug. If you don't know pug, read the primer on the website.

Now you should see your first page on http://localhost:3000/.

Make your page editable

Nice, a static page is useful, but we could haved reached to that point easily without penguin. The real power comes in if we make the content on our page editable. To accomplish this we need so-called components which will be described in detail later. For now we just install two components using the following command ...

npm i -S penguin-inplace
... and list them in our package.json.
package.json
{
  "penguin": {
    "languages": ["en"],
    "components": {
      "Inplace": "penguin-inplace",
      "SaveButton": "penguin-save-button"
    }
  },
  "scripts": {
    "serve": "penguin serve -v pug",
    "prebuild": "penguin pack -v pug",
    "build": "penguin build"
  }
}

Optimisation coming up!

I'm planning a penguin install command to install new components having to manually touch the package.json. Stay tuned, I'll roll this out in the next weeks.

Now we can use the inplace component in our page to add editable fields. Add the following pug snippet to your page body.

span(data-component='Inplace' data-props='{"field":"my-first-field"}') My first editable content

If you refresh the page, you'll see that the inplace component transforms your element into a content-editable element. Great, but now we want to be able to persist our changes. For that we use the save-button component:

button(data-component='SaveButton') Save

If you now refresh, change your editable field and click the button, penguin will store the content. Check that by refreshing your page.
